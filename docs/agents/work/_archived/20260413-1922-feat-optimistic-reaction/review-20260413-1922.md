### 問題点

1. **Optimistic UIが動作していない**: `emojiMenu/module.tsx` の epic ハンドラ内で `SlackActions.onReactionAdded({...})` / `SlackActions.onReactionRemoved({...})` を呼んでいるが、これはアクションオブジェクトを生成するだけで **ディスパッチされていない**。typeless の `createModule` で生成されるアクションクリエータは、呼び出すだけではストアにディスパッチしない。正しくディスパッチするには以下のいずれかが必要:
   - `appRegistry.dispatch(SlackActions.onReactionAdded({...}))` で明示的にディスパッチする（`rtm-socket.ts` で使われているパターン）
   - epic のハンドラの戻り値としてアクションを返す（`slack/module.ts` の epic で使われているパターン。ただし `await` 前に即時ディスパッチしたい場合は `appRegistry.dispatch` が適切）

   根拠: コンソールログに `slack/ON_REACTION_REMOVED` が一度も出力されず、`emojiMenu/REMOVE_REACTION` → `[mock] removeReaction` の流れのみ確認された。サーバー経由の `slack/ON_REACTION_ADDED` は RTM イベント経由で正しくディスパッチされていた。
