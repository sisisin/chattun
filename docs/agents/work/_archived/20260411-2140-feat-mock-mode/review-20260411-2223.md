### 指摘事項

1. **[server/mock-server/presets.ts:113-123]**: `message-deleted` プリセットで `ts` フィールドに `nowTs()` を2回呼び出しており、`deleted_ts` と `ts` が異なるタイムスタンプになる。Slack の実際のイベントでは `ts` は削除通知自体のタイムスタンプ、`deleted_ts` は削除されたメッセージのタイムスタンプであるため意図的に異なる値にする設計は理解できるが、コード内で変数 `ts` に代入した値を `deleted_ts` に使い、`ts` には新たな `nowTs()` 呼び出しを使っており、変数名と実際の使い方が逆転していて紛らわしい。変数名を `deletedTs` に変更するか、コメントで意図を明示することを推奨する。

   ```ts
   // 修正例
   'message-deleted': () => {
     const deletedTs = nowTs();
     const ts = nowTs();
     return {
       type: 'message',
       subtype: 'message_deleted',
       channel: MOCK_CHANNEL,
       deleted_ts: deletedTs,
       ts,
       event_ts: deletedTs,
     };
   },
   ```

2. **[front/src/app/services/http/MockSlackClient.ts:5-11]**: モジュールレベルの変数 `bootstrapCache` が `any` 型で定義されており、型安全性が欠如している。モックモードはデバッグ・開発目的のコードであるが、`fetchBootstrap()` の戻り値に適切な型を付けることで誤ったプロパティアクセスを防げる。少なくとも `{ emojis: SlackAPI.Emoji.List; users: SlackAPI.Users.List; channels: SlackAPI.Conversations.List; teamInfo: SlackAPI.Team.Info }` のような型を定義することを推奨する。

3. **[front/src/app/services/http/MockSlackClient.ts:7-11]**: `fetchBootstrap()` 関数で `res.ok` チェックがなく、モックサーバーが起動していない場合や `/api/mock/bootstrap` が404を返した際に `res.json()` でエラーとなるか不正なデータがキャッシュされる。エラー時には明示的なエラーメッセージを出すと開発体験が改善される。

   ```ts
   async function fetchBootstrap() {
     if (bootstrapCache) return bootstrapCache;
     const res = await fetch('/api/mock/bootstrap');
     if (!res.ok) {
       throw new Error(`[mock] /api/mock/bootstrap の取得に失敗しました: ${res.status}`);
     }
     bootstrapCache = await res.json();
     return bootstrapCache;
   }
   ```

4. **[server/mock-server/index.ts:117-118]**: `app.get('/api/*', ...)` の404ハンドラーは GET リクエストのみを対象にしており、存在しない `/api/mock/event/xxx` への POST リクエストなどは404にならず SPA フォールバックにルーティングされる。GET 以外の未定義 API リクエストも適切に404を返すよう `app.all('/api/*', ...)` に変更することを推奨する。

   ```ts
   // 修正例
   app.all('/api/*', (_req, res) => {
     res.status(404).end();
   });
   ```

5. **[README.md]**: モックモードの起動手順が README に記載されていない。`yarn dev:mock` コマンドや `REACT_APP_MOCK_MODE=true` 環境変数の設定方法など、モックモードを使うための手順を README または CLAUDE.md に追記することを推奨する。これがないと開発者がモックモードの存在に気づけない。
