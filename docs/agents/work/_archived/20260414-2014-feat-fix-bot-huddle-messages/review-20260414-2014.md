STATUS: NEEDS_FIX

### 指摘事項

1. **[front/src/app/features/slack/selector.ts:99-102]**: `permalink` は `https://` で始まるウェブURLだが、`deepLinking: 'directly'` のときも `type: 'directly'` がそのまま返される。`DeepLinkingButton` は `type === 'directly'` の場合は `target="_blank"` を付けない（`slack://` プロトコル前提）。その結果、ハドルメッセージのリンクが `directly` モードで `https://` URLを `target` なし `href` として使うことになり、期待する挙動にならない。

   `permalink` を使う場合は常に `viaBrowser` 相当の扱い（`target="_blank"`）にすべき。修正例:

   ```ts
   if (msg.permalink) {
     return { type: 'viaBrowser', link: msg.permalink };
   }
   ```
