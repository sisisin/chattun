## Browser Verification

STATUS: OK

### 確認内容

- **CopyButton「C」ボタンのtoast表示**: 設定画面から開発者モードを有効にし、モックメッセージを送信してタイムラインにメッセージを表示。「C」ボタンクリック後、MutationObserverにより `<div class="_toast_erlro_1">Copied!</div>` がDOMに追加されることを確認。typelessのコンソールログでも `toast/SHOW_TOAST` アクションが正常に発火している。
- **ボタン文言が「C」のまま変化しないこと**: 複数回クリック後もボタンの文言は「C」のまま変化しないことをスクリーンショットで確認。旧実装の `copied` state による文言変更は完全に削除されている。
- **toastの2秒後自動消去**: MutationObserverのタイムスタンプにより、toast要素がDOMに追加されてから約2.6秒後（TOAST_DURATION=2000ms + RxJS delay + React再レンダリング遅延）に自動削除されることを確認。コンソールログでも `toast/SHOW_TOAST` から約2秒後に `toast/HIDE_TOAST` が発火している（例: 20:34:23 -> 20:34:25）。
- **コンソールエラー**: エラー・例外は検出されなかった。

### 補足

ブラウザ自動化ツールのスクリーンショットのタイミング遅延により、toast表示中のスクリーンショットキャプチャは困難だったが、MutationObserverとtypelessのアクションログの両方で以下を裏付けるデータを取得した:
1. クリック時に `toast/SHOW_TOAST` アクションが発火し、toastストアの `message` が `"Copied!"` に更新される
2. ReactのcreatePortalにより `document.body` にtoast要素（`<div class="_toast_erlro_1">Copied!</div>`）が追加される
3. 約2秒後に `toast/HIDE_TOAST` アクションが発火し、toast要素がDOMから削除される
