## Round 1

STATUS: NEEDS_FIX

### 指摘事項

1. **[front/src/app/imports.ts 削除]**: `imports.ts`を削除したことにより、`emoji-mart`の`init({ data })`呼び出しが失われている。これは**機能バグ**である。

   `init`は絵文字ピッカー(`<Picker>`)ではなく、Web Componentの`<em-emoji>`タグのために必要な初期化処理。`Tweet.tsx:73`で`<em-emoji id={elem.name} size="20px" />`が使われており、これはSlackのリアクション表示（カスタム絵文字以外の標準絵文字）に使われている。

   `init`なしでは`<em-emoji>`が絵文字を表示できず、リアクション表示が壊れる。

   **修正案**: `@emoji-mart/data`をそのまま初期バンドルに残したくないなら、`LazyPicker`のdynamic importと同様に`init`呼び出しも遅延化（または別の方法で初期化）する必要がある。たとえばアプリ起動後に非同期で`init`を呼ぶ処理を追加する。

   ```ts
   // 例: index.tsxやApp.tsxで
   import('@emoji-mart/data').then(({ default: data }) => {
     import('emoji-mart').then(({ init }) => {
       init({ data });
     });
   });
   ```

   ただし、この場合は初期バンドルへの`@emoji-mart/data`の混入を避けるため、既存の`LazyPicker`と`init`の動的インポートを同一チャンクに乗せる（`EmojiMenuView.tsx`内で`init`も呼ぶ等）か、アプリ起動直後に非同期で呼ぶか、設計を明確にした上で実装する必要がある。

2. **[front/src/app/features/emojiMenu/components/EmojiMenuView.tsx:13]**: `LazyPicker`内部のインライン関数コンポーネントの`props`型に`any`が使われている（`custom: any`）。既存のコードベースにてany使用が許容されているか確認の上、可能であれば`CustomEmojiInfo[][]`等の具体型を使用すること。

   ```ts
   // 改善案
   import { CustomEmojiInfo } from 'app/features/timeline/interface';
   
   (props: { custom: CustomEmojiInfo[][]; onEmojiSelect: (emoji: { id: string }) => void })
   ```
