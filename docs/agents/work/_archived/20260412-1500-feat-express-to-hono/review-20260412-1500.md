### 指摘事項

1. **[server/src/index.ts:82-106]: OAuth リダイレクト失敗時のレスポンス二重書き込み問題（重大）**

   `installer.handleCallback` の `failure` コールバック内で `outgoing.writeHead(500); outgoing.end('OAuth failed')` を直接呼んでいる。`handleCallback` はその後例外をthrowせず正常に解決するため、`installation === null` となり `return c.body(null)` が実行される。Hono の node-server アダプター (`responseViaResponseObject`) は `outgoing.headersSent` を確認せずに `outgoing.writeHead(res.status, ...)` を呼ぼうとするため、`"Cannot set headers after they are sent to the client"` エラーが発生する。

   同様に catch ブロック (90-97行) でも `outgoing.writeHead(500); outgoing.end('OAuth failed')` を書き込んだ後に `return c.body(null)` を返しているため、同じ問題が発生する。

   **修正案**: `failure` コールバックで `outgoing` に直接書き込む代わりに、Hono のレスポンス機構を使用する。または `failure` コールバックで書き込んだ後は Hono にレスポンスを書き込ませないよう `c.header('x-hono-already-sent', '1')` を設定してから返す。より明確な修正例:

   ```typescript
   app.get('/api/slack/oauth_redirect', async c => {
     const { incoming, outgoing } = c.env;
     let installation: any = null;
     let oauthError: Error | null = null;

     try {
       await installer.handleCallback(incoming, outgoing, {
         success: inst => {
           installation = inst;
         },
         failure: (error) => {
           oauthError = error as Error;
         },
       });
     } catch (err) {
       logger.errore('OAuth callback error', err);
       return c.text('OAuth failed', 500);
     }

     if (oauthError) {
       return c.text('OAuth failed', 500);
     }

     if (installation) {
       const sessionId = createSessionId();
       await saveSession(sessionId, { slack: installation });
       c.header('Set-Cookie', serializeSessionCookie(sessionId));
       return c.redirect(serverBaseUrl, 302);
     }

     return c.body(null, 400);
   });
   ```

2. **[server/src/session.ts:16-19, server/src/io.ts:13-14]: クッキーパース実装の重複**

   `session.ts` に `parseSessionIdFromCookie` という生の正規表現パース実装と、`sessionMiddleware` での `getCookie(c, SESSION_COOKIE)` (hono/cookie) の2種類が存在する。`parseSessionIdFromCookie` は Socket.IO の `socket.request.headers.cookie` に対して使われているが、正規表現のエッジケース（クッキー値にURLエンコードが含まれる場合など）で `getCookie` と挙動が異なる可能性がある。

   **修正案**: `parseSessionIdFromCookie` の代わりに Node.js 標準の `cookie` モジュールまたは hono の内部ユーティリティを使用してクッキーをパースするか、少なくとも実装を統一する。
