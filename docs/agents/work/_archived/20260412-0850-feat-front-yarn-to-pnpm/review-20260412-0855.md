STATUS: NEEDS_FIX

### 指摘事項

1. **[mise.toml]**: `yarn = "1"` が削除されているが、`server/` は引き続き yarn 1.x を使用している（`server/package.json` に `"yarn": "^1.22.5"` が残っており、`CLAUDE.md` にも「server は yarn 1.x」と記載）。Node 22.x に同梱されている corepack 経由の yarn でローカル動作する可能性はあるが、mise でバージョンが固定されなくなることで環境差異が生じるリスクがある。`yarn = "1"` を復活させるか、もしくはサーバーが corepack 管理の yarn で問題なく動作することを確認・ドキュメント化すべき。

2. **[.github/workflows/pull--check.yaml]**: pnpm へ移行したにもかかわらず、`actions/setup-node` に `cache: 'pnpm'` と `cache-dependency-path: 'front/pnpm-lock.yaml'` が設定されていない。旧 yarn 設定は削除されたが、pnpm 相当のキャッシュが追加されていないため、毎回フルインストールが走りCIが遅くなる。以下を追加することで対処できる:
   ```yaml
   cache: 'pnpm'
   cache-dependency-path: 'front/pnpm-lock.yaml'
   ```

3. **[front/.gitignore]**: yarn 関連エントリ (`yarn-debug.log*`, `yarn-error.log*`) が残っている一方、pnpm 相当のログファイル (`pnpm-debug.log*`) が追加されていない。機能的な問題はないが、`.gitignore` の整合性のために `pnpm-debug.log*` を追加し、yarn 関連エントリを削除することを検討すべき。
