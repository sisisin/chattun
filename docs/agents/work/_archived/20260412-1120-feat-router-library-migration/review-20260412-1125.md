## Round 1

STATUS: NEEDS_FIX

### 指摘事項

1. **[front/src/app/hooks/useRouter.ts:5]**: `useParams({ strict: false })` の呼び出しに型エラーがある。`npx tsc --noEmit` を実行すると以下のエラーが発生する:
   ```
   Argument of type '{ strict: false; }' is not assignable to parameter of type 'UseParamsOptions<...>'
   ```
   タスク達成条件に「型チェックがパスする」とあるが、現在の実装では型チェックが通らない。`structuralSharing` オプションの追加か、`{ strict: false } as any` で回避するのではなく、TanStack Router の正しい呼び出し方に修正する必要がある。例えば `useParams({ strict: false, structuralSharing: false })` などを試みるか、`from` を省略して正しいオーバーロードを使う方法を検討すること。

2. **[front/src/app/features/thread/module.tsx:31]**: `useRouter<ThreadParam>()` に型エラーがある。`ThreadParam` は `{ channelId: string; ts: string }` というインターフェースで、インデックスシグネチャ (`[key: string]: string`) がないため `Record<string, string>` の制約を満たさない。
   ```
   Type 'ThreadParam' does not satisfy the constraint 'Record<string, string>'.
   Index signature for type 'string' is missing in type 'ThreadParam'.
   ```
   `useRouter.ts` の型制約 `T extends Record<string, string>` が `ThreadParam` と互換性がない。`useRouter.ts` の型制約を `T extends Record<string, string> = Record<string, string>` のように緩めるか、`ThreadParam` に `[key: string]: string` インデックスシグネチャを追加する必要がある。

   なお上記 1 と 2 は連動しており、根本的には `useParams({ strict: false })` の型制約を正しく扱うことで両方修正できる可能性が高い。

3. **[front/src/app/components/AppLink.tsx:5-6]**: `to` プロパティの型が `string` のため、TanStack Router の型安全性が活かせていない。`Link` コンポーネントはルートツリーに登録されたパスのみを受け付ける型制約を持つが、`AppLink` の `to: string` はこの恩恵を受けられない。型チェック上の問題ではないが、TanStack Router 移行の主目的の一つ「フルタイプセーフなルーティング」を活かせていない点は留意すること。ただし既存の利用箇所が限定的で動作上の問題はないため、改善提案として記載する。
