**エラー内容:**
```
TypeError: has is not a function
    at RenderNode (MrkdwnRenderer.tsx:146:8)
```

**根本原因:**
`front/src/app/features/mrkdwn/MrkdwnRenderer.tsx:1` で `node-emoji` から `has` をインポートしているが、`node-emoji` v1.10.0 には `has` という名前のexportが存在しない。v1.10.0 の実際のexportは `hasEmoji` である。

```typescript
// 現在のコード（壊れている）
import { emojify, has } from 'node-emoji';

// 正しくは
import { emojify, hasEmoji } from 'node-emoji';
// かつ 65行目の has(node.name) を hasEmoji(node.name) に修正
```

`node-emoji` v2.x では `has` がexportされているが、このプロジェクトにインストールされているのは v1.10.0 であるため、`hasEmoji` を使う必要がある。

### 問題点

1. **[CRITICAL] `has is not a function` ランタイムエラー**: `MrkdwnRenderer.tsx:65` の `has(node.name)` 呼び出しが `TypeError` を発生させ、メッセージ表示が完全にクラッシュする。`node-emoji` v1.10.0 では `has` ではなく `hasEmoji` がエクスポートされている。インポート文と呼び出し箇所の両方を `hasEmoji` に修正する必要がある。
   - **影響範囲**: emoji ノードを含むすべてのメッセージが表示不能になる。emoji ノードを含まないメッセージもError Boundaryの巻き添えで表示されなくなる。
   - **修正箇所**: `front/src/app/features/mrkdwn/MrkdwnRenderer.tsx` の1行目と65行目

**注意**: このクラッシュにより、Round 1 で指摘された画像添付表示やその他の検証項目を確認することができなかった。`has` の修正後に再度ブラウザ検証を行う必要がある。

**エラー内容:**
```
TypeError: has is not a function
    at RenderNode (MrkdwnRenderer.tsx:146:8)
```

**根本原因（更新）:**
Round 2 では `node-emoji` v1.10.0 がインストールされているためと報告したが、実際には **v2.2.0 はすでにインストール済み**である（`front/node_modules/node-emoji` → `node-emoji@2.2.0` へのシンボリックリンクを確認済み）。v2.2.0 のソースコード（`lib/index.js`）には `has` が正しくnamed exportされている。

しかし、**Viteの依存関係プリバンドルキャッシュ**（`front/node_modules/.vite/deps/node-emoji.js`）が **古い v1.10.0 のコードをバンドルしたまま**残っている。キャッシュファイルの末尾に以下の記述がある:

```
// ../node_modules/.pnpm/node-emoji@1.10.0/node_modules/node-emoji/index.js
export default require_node_emoji();
```

v1.10.0 は CommonJS + default export のみであり、named export の `has` が存在しないため、`import { has } from 'node-emoji'` が `undefined` を返しクラッシュする。

### 問題点

1. **[CRITICAL] Viteプリバンドルキャッシュの不整合**: `node-emoji` は v2.2.0 にアップグレード済みだが、Viteのプリバンドルキャッシュ（`front/node_modules/.vite/deps/node-emoji.js`）が v1.10.0 のバンドル結果を保持している。これにより `import { has } from 'node-emoji'` が `undefined` を返し、ランタイムで `TypeError: has is not a function` が発生する。

   **修正方法**: Viteのdepsキャッシュを削除してdevサーバーを再起動する。
   ```sh
   rm -rf front/node_modules/.vite/deps
   # devサーバーを再起動
   ```
   または `front/node_modules/.vite` ディレクトリ全体を削除する。Viteは起動時に自動で再プリバンドルを行う。

**注意**: キャッシュ問題の修正後に、全ての検証項目（メッセージ表示、絵文字、メンション、リンク、画像、コードブロック、フォーマット）を再度確認する必要がある。
