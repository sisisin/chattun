## Browser Verification

STATUS: BLOCKED

### 確認内容

- devサーバー（port 3000）: 起動中（HTTPS モード、自己署名証明書 `local.sisisin.house`）
- モックサーバー（port 3100）: 起動中、正常応答を確認（curl でプリセットイベント送信成功）
- ブラウザ拡張（claude-in-chrome）: 接続済み、タブ操作・JS実行は正常動作
- `https://local.sisisin.house:3000` へのアクセス: Chrome のプライバシーエラー（NET::ERR_CERT_AUTHORITY_INVALID 相当）

### 問題点

1. Vite devサーバーが HTTPS モード（`tmp/fullchain.pem` / `tmp/privkey.pem`、自己署名証明書 CN=local.sisisin.house）で起動しており、Chrome がこの証明書を信頼していないためプライバシーエラーが表示される
2. Chrome の interstitial（プライバシーエラー）ページは CDP でのアタッチが不可能なため、`thisisunsafe` キーワード入力や「詳細設定→このサイトにアクセスする」操作をプログラム的に実行できない
3. HTTP（`http://localhost:3000`）ではサーバーが empty reply を返す（HTTPS のみ対応のため）

### 解決に必要な対応

以下のいずれかで再検証可能:
- Chrome で手動で `https://local.sisisin.house:3000` にアクセスし、「詳細設定」→「local.sisisin.house にアクセスする（安全ではありません）」を一度クリックして証明書例外を追加する
- `tmp/fullchain.pem` の CA 証明書を macOS キーチェーンに信頼済みとして登録する（`security add-trusted-cert`）
- mkcert 等で作成した証明書をシステムに信頼させる
