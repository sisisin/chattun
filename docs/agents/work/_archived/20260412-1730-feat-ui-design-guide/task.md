# Task: UIデザインの設計を明文化する

## 達成条件

- docs/guide/ui-design.mdにタイポグラフィや色味（primary/secondary colorなど）が最低限まとめてある状態

## 背景

UIに手を加えていくにあたり、現在のデザインの設計意図が明文化されていない。
ソースコードを読み、画面のデザイン・色味・アイコンなどを解釈して設計意図を分析し、今後守っていくべきデザインのためのドキュメントを構築する。

## 進め方

- CSSファイル（main.css, 各コンポーネントCSS）を読み解く
- CSS変数・カラーコード・フォント設定を抽出
- ダーク/ライトモードの対応状況を確認
- コンポーネントの視覚パターンを整理
