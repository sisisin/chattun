### 指摘事項

1. **[docs/guide/ui-design.md:71-75 タイポグラフィ > フォント]**: 記載されているフォントファミリーが実際のソースコードと一致しない。ドキュメントには `Noto Sans`, `Apple Color Emoji`, `Segoe UI Emoji` 等が含まれているが、実際の `_base.css` の body 定義は以下の通り：
   ```css
   font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 'Oxygen', 'Ubuntu', 'Cantarell', 'Fira Sans', 'Droid Sans', 'Helvetica Neue', sans-serif;
   ```
   ガイドに記載されているフォントスタックはリポジトリのどのCSSファイルにも存在しない。実際のソースの値に修正すること。

2. **[docs/guide/ui-design.md:145 アイコン]**: アイコンコンポーネントのファイルパスが `app/components/icons/Icons.tsx` と記載されているが、基準となるディレクトリが不明確。実際のパスは `front/src/app/components/icons/Icons.tsx` であるため、リポジトリルートからの相対パスで `front/src/app/components/icons/Icons.tsx` と記載するか、参照ルートを明示すること。

3. **[docs/guide/ui-design.md:63-65 シャドウ]**: 「5段階（weakest → strongest）」とのみ記述されており、各トークン名が省略されている。実際には `--shadow-weakest`, `--shadow-weak`, `--shadow-normal`, `--shadow-strong`, `--shadow-strongest` の5つが定義されている。新しいUIを実装する際に参照できるよう、他のセクションと同様にトークン名を表形式または一覧で記載すること。

4. **[docs/guide/ui-design.md:39-62 セマンティックトークン表]**: ライトモード・ダークモードの値の列で `light-main`, `grayscale00` のように `--` プレフィックスなしで記載されているが、これらはCSS変数名（`var(--light-main)` 等）の省略表記として読むことができる。参照時の混乱を避けるため、「値はCSSカスタムプロパティ名（`--` 付き）で参照すること」などの注記を追加するか、表記を統一すること。

5. **[docs/guide/ui-design.md:152-154 ダークモード]**: 実際の `_colors.css` ではライトモード用の `@media (prefers-color-scheme: light)` ブロックがコメントアウトされており、現在はデフォルト（`:root`）でライトモードの値が定義される構造になっている。このコメントアウト状態はダークモード実装の「想定された移行状態」を示しているが、ドキュメントにはこの実装の現状が反映されていない。「現在はデフォルト値としてライトモード用の値が定義されており、`prefers-color-scheme: dark` のみメディアクエリで上書きされる」という実装の実態を記載すること。
