### 指摘事項

1. **[front/src/app/features/session/module.ts:22-23]**: `appHistory.push('/login')` でのリダイレクトとToast表示の順序が問題。`appHistory.push('/login')` を先に呼び出した後に `ToastActions.showToast(...)` をdispatchするため、ページ遷移がToastの表示より先に発火する。Toastは `createPortal` で `document.body` に描画されるためルート変更後も表示自体はされるが、Toast表示のdispatchはepicのreturnで行われる（typelessのアクション処理フロー内）。一方 `rtm-socket.ts` 側は `appRegistry.dispatch` でToastを先にdispatchしてから `appHistory.push('/login')` を呼ぶ順序になっており（16行目 → 22行目）、両者で順序が異なっている。

   タスクの達成条件「ユーザー体験が損なわれない（メッセージを見てからリダイレクトされる）」を考えると、`session/module.ts` では `ToastActions.showToast` のdispatchが先にstateに反映されてからリダイレクトする必要がある。しかし現在の実装では `appHistory.push('/login')` がToast表示のdispatch（epicのreturn）より先に実行される。

   `rtm-socket.ts` と同様に、Toast dispatchを先に行うよう修正すべき。例えば以下のように変更する：

   ```ts
   appRegistry.dispatch(ToastActions.showToast(`${msg}再度ログインしてください。`));
   appHistory.push('/login');
   return null;
   ```

   または、typelessのepic戻り値でToastをdispatchする現在の方式を維持する場合でも、`appHistory.push('/login')` をepicの戻り値の後（typeless内部でdispatch後）に呼ぶよう設計を見直す必要がある。
