### 指摘事項

1. **[front/src/app/features/setting/module.tsx:25-30]**: 短時間に複数回設定変更した場合のタイマー競合バグ。`updateSettingFulfilled` のEpicは `mergeMap` ベースで動作するため、設定を連続変更すると複数の `delay(2000)` タイマーが並行して走る。最初のタイマーが2秒後に `hideToast` を発行すると、2回目の保存から2秒経過していなくてもトーストが消えてしまう。`switchMap` を使って前のタイマーをキャンセルするか、または Epic の代わりに `useEffect` でタイマーを管理する実装に変更すること。

   ```tsx
   // 修正例: switchMapを使う
   import { switchMap } from 'rxjs/operators';

   .on(SettingActions.updateSettingFulfilled, (_, { action$ }) => {
     return action$.pipe(
       switchMap(() => of(null).pipe(
         delay(2000),
         map(() => SettingActions.hideToast()),
       )),
     );
   });
   ```

   ただし typeless の `.on()` APIでは上記の書き方はできないため、Epicの外でRxJSを直接使うか、あるいはReactの `useEffect` でタイマー管理をする方法（`showToast` が `true` になったら `setTimeout` でセットし、`showToast` が変わるたびにクリアする）が現実的。
