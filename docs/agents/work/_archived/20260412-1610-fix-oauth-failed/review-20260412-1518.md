## Round 1

STATUS: NEEDS_FIX

### 指摘事項

1. **[server/src/slack.ts:16]**: `stateSecret` が `'my-state-secret'` というハードコードされた固定値になっています。`stateSecret` は CSRF 検証用の state パラメータを署名するために使われており、外部に漏れると攻撃者が正規の state を偽造できます。`config.ts` から環境変数経由で読み込む形に変更してください。例: `stateSecret: slackStateSecret` として `config.ts` に `SLACK_STATE_SECRET` 環境変数を追加する。

   なお、この問題は今回の変更で新規導入されたものではなく元々存在していた可能性もありますが、`handleInstallPath` への切り替えにより state CSRF 検証が実際に機能するようになったことで、この値の安全性がより重要になっています。

2. **[server/src/index.ts:95-98]**: `handleCallback` の `failure` コールバックで `failureError` に格納されたエラーは `Error` 型とは限りません。`@slack/oauth` の `failure` コールバックの型シグネチャは `(error: CodedError) => void` ですが、`CodedError` は `Error` を継承しているため実際の型問題はありません。ただし `failureError` の変数宣言が `Error | null` になっているので、より正確に `import type { CodedError } from '@slack/oauth'` を使った型にすることで意図が明確になります（軽微な改善提案）。

   **必須修正ではなく、指摘事項1が主要な問題です。**

### Dismissed

- **指摘1 (stateSecret ハードコード)**: 今回の変更で導入された問題ではなく、既存コードの問題。このタスクのスコープは「OAuth failedの修正」であり、stateSecretの環境変数化は別タスクとして TODO.md に追加する。
- **指摘2 (CodedError型)**: レビューアー自身が「必須修正ではない」と記載。`Error`を継承しているため実用上の問題なし。
