# Task: 設定フォームのUX改善

## 背景

設定画面の Channel Match / ミュート設定で以下のUX問題がある:
1. バツボタンがまだ少し小さい
2. 行ごとに保存ボタンがありチグハグ（バツボタンを押すと「保存しました」が出る等）
3. フォームが空・変更なしでも保存ボタンが有効
4. トースト「保存しました」が連続発生時にフィードバックが消える（同じメッセージだとUIが変化しない）

## 設計

### 保存ボタンの統一
- Channel Match / ミュート の各セクションで、行ごとの保存ボタンを削除
- セクション下部に1つの「保存」ボタンを配置
- 削除（✕）は即時反映（ローカルstateとストア両方を更新）、保存ボタン不要
- 追加（+）はローカルstateのみ更新、保存ボタンでストアに反映

### 保存ボタンの disable
- ローカルstate と ストアの値を比較し、差分がなければ disabled
- 空のmatchValue/空文字ユーザーを含む行がある場合も disabled（不完全なデータの保存防止）

### バツボタンサイズ
- 現在36px。input/selectと同程度の高さにするため、padding付きで自然な大きさにする

### トースト連続問題
- showToast 時にversion をトーストのkeyとして使い、同じメッセージでもコンポーネントが再マウントされてアニメーションが再開するようにする

## 達成条件

1. Channel Match / ミュート の行ごとの保存ボタンを廃止し、セクション下部に1つの保存ボタンを配置
2. 変更がない・空の入力がある場合に保存ボタンが disabled
3. トーストが連続発生時に再度視覚フィードバックがある（key によるリマウント）
4. vp check が通ること
